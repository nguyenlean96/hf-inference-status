### Planner Response

Great question! **Lazy** in Polars is about **when** computation happens.

## Eager vs Lazy

### Eager (Default)
```rust
// Each operation executes IMMEDIATELY
let df = df.filter(...)?;     // runs now
let df = df.select(...)?;     // runs now
let df = df.group_by(...)?;   // runs now
```

Each line blocks until complete. If you chain many operations, they run one-by-one.

---

### Lazy
```rust
// Build a PLAN, nothing runs yet
let lf = df.lazy()
    .filter(...)      // just adds to plan
    .select(...)      // just adds to plan
    .group_by(...)    // just adds to plan
    .sort(...);       // just adds to plan

// NOW it runs - optimized!
let result = lf.collect()?;
```

## Why Lazy is Better for Your Use Case

Since you're doing **multi-column sorting + multi-column filtering**, lazy gives you:

| Benefit                              | What It Means                                                                              |
| ------------------------------------ | ------------------------------------------------------------------------------------------ |
| **Query optimization**               | Polars rearranges your operations for speed (e.g., filter before sort = less data to sort) |
| **Predicate pushdown**               | Filters move as early as possible                                                          |
| **Projection pushdown**              | Only reads columns you actually need                                                       |
| **Common subexpression elimination** | Reuses repeated calculations                                                               |

## Simple Analogy

Think of it like a **shopping list** vs **individual trips**:

- **Eager**: Go to store → buy eggs → come home → go to store → buy milk → come home → ... (inefficient)
- **Lazy**: Write list → go once → buy everything → come home (optimized)

---

```rust
    pub provider: Option<Vec<String>>,   // contains any of these
    pub tools: Option<bool>,             // None = no filter
    pub structured: Option<bool>,        // None = no filter
}

pub fn query_models(
    df: &DataFrame,
    filters: FilterQuery,
    sorts: Vec<SortColumn>,
) -> PolarsResult<DataFrame> {
    let mut lf = df.clone().lazy();

    // Apply filters
    if let Some(models) = filters.model {
        // Filter: model contains any of the search terms
        let expr = models.iter()
            .map(|m| col("model").str().contains(lit(m.as_str()), false))
            .reduce(|a, b| a.or(b))
            .unwrap_or(lit(true));
        lf = lf.filter(expr);
    }

    if let Some(tools) = filters.tools {
        lf = lf.filter(col("tools").eq(lit(tools)));
    }

    // Apply multi-column sort
    let sort_exprs: Vec<_> = sorts.iter()
        .filter(|s| s.order.is_some())
        .map(|s| col(&s.column))
        .collect();
    
    let descending: Vec<_> = sorts.iter()
        .filter(|s| s.order.is_some())
        .map(|s| s.order.as_ref().map(|o| o == "desc").unwrap_or(false))
        .collect();

    if !sort_exprs.is_empty() {
        lf = lf.sort_by_exprs(
            sort_exprs,
            SortMultipleOptions::default().with_order_descending_multi(descending)
        );
    }

    lf.collect()
}
```
